# بوّابة الترقي — مع المساعد الذكي


## التعديلات الجديدة في هذا الإصدار
1. **بحث الويب (Tavily) — اختياري**: في `worker.js`، أضف متغيراً ثانياً
   `TAVILY_API_KEY` بنفس طريقة `GROQ_API_KEY` (Settings ← Variables ← Add ←
   Encrypt) إن رغبت تفعيله. يُستخدم فقط حين يبدو السؤال متعلقاً بمعلومة
   حديثة (تواريخ، "2025/2026"، "قرار جديد"...)، وليس على كل سؤال — توفيراً
   للاستهلاك. بدون هذا المفتاح، يعمل المساعد بشكل طبيعي عبر قاعدة المعرفة
   ومعرفة Groq العامة فقط.
2. **أسلوب المساعد**: أصبح داعماً ومحفزاً صريحاً في لهجته، مع تشديد إضافي
   على الدقة (يصرّح بعدم التأكد بدل تخمين أرقام مراسيم أو تواريخ) — نظراً
   لحساسية الموضوع.
3. **الهيدر**: استُبدلت الجملة التعريفية بحقوق الملكية: "© DONIA LABS TECH
   — تطوير: فريق DONIA LABS TECH | قائد المشروع: رائد الأعمال الأستاذ طوينة
   داود".
4. **القانون الأساسي الجديد لسنة 2025**: تم البحث والتحقق من المصدر الرسمي،
   وإضافة درس جديد + سؤال تقييمي + مقتطف في قاعدة المعرفة عن **المرسوم
   التنفيذي 25-54 المؤرخ في 21 جانفي 2025** (وما يرافقه من المرسوم 25-55
   الخاص بالتعويضات)، والذي عوّض الإطار السابق (08-315 و12-240). تم تضمين
   تنبيه صريح بأن بعض تعديلاته المتعلقة برتب الإدارة كانت لا تزال قيد
   التشاور مع النقابات حتى مطلع 2026 ولم تُنشر بصيغتها النهائية — يُنصح
   المترشح بالتحقق دائماً من آخر نسخة رسمية قبل الامتحان.

هذا تعديل على ملفك الأصلي (`mudir-thanawiya-2026.html`) مع إضافتين:

1. **تبويب "🤖 المساعد الذكي"** جديد، يجيب بالاستناد إلى قاعدة معرفة حقيقية
   مستخرجة من أرشيفك (`data/knowledge.json`) — وثائق التشريع والتربية
   المختصرة + نماذج امتحانات سابقة فعلية (2000-2004 وغيرها).
2. **إصلاح صامت مهم**: الملف الأصلي كان يستخدم `window.storage`، وهي ميزة
   خاصة بمعاينة Claude.ai فقط ولا تعمل خارجها. تم استبدالها بـ
   `localStorage` العادي، فسجل تقدمك يعمل الآن بشكل طبيعي على أي استضافة
   (GitHub Pages وغيرها).

## الملفات في هذه الحزمة
```
index.html          ← الملف الرئيسي (نفس تطبيقك + التبويب الجديد)
data/knowledge.json  ← قاعدة المعرفة (984 مقتطفاً من 113 وثيقة)
worker.js            ← خادم وسيط صغير لإخفاء مفتاح Groq (يُنشر على Cloudflare لا GitHub)
```

## لماذا يوجد ملف `worker.js` منفصل؟
GitHub Pages يعرض ملفات فقط، بدون أي خادم خلفي. لو وضعنا مفتاح Groq داخل
`index.html` مباشرة، سيظهر لأي شخص يفتح الصفحة (حتى بدون أدوات تقنية،
بمجرد "عرض مصدر الصفحة"). الحل الأسرع والمعتمد هنا: واسطة صغيرة جداً (ملف
واحد) تُنشر على Cloudflare مجاناً، تحتفظ بمفتاحك سرّاً، والصفحة على GitHub
Pages تتحدث معها فقط. أنت تُدخل المفتاح مرة واحدة هناك، والمستخدم يستعمل
التطبيق مباشرة دون أي إدخال — وهو تماماً ما طلبته.

## خطوات التشغيل

### 1) نشر `worker.js` على Cloudflare (٢ دقيقة)
1. dash.cloudflare.com ← **Workers & Pages** ← **Create** ← **Worker** (وليس Pages).
2. امسح الكود الافتراضي، وألصق محتوى `worker.js` كاملاً ← **Deploy**.
3. من صفحة الـ Worker ← **Settings** ← **Variables** ← **Add variable**:
   - الاسم: `GROQ_API_KEY`
   - القيمة: مفتاحك
   - فعّل **Encrypt** ← **Save and deploy**.
4. انسخ رابط الـ Worker الظاهر أعلى الصفحة (شكله
   `https://xxxx.yyyy.workers.dev`).

### 2) ربط الرابط بالتطبيق
افتح `index.html`، ابحث عن السطر التالي قرب نهاية الكود:
```js
const WORKER_URL = "https://YOUR-WORKER-NAME.YOUR-SUBDOMAIN.workers.dev";
```
وضع رابطك الحقيقي مكانه. هذا الرابط نفسه غير سرّي (لا بأس أن يظهر في الكود) —
السرّ الوحيد هو مفتاح Groq المحفوظ داخل الـ Worker فقط.

### 3) رفع المشروع على GitHub Pages
1. github.com ← **New repository** ← اسم مثلاً `mudir-thanawiya-2026` ← **Public** ← أنشئه.
2. **uploading an existing file** ← اسحب `index.html` ومجلد `data` كاملاً معه (لا حاجة لرفع `worker.js` هنا، فهو يعيش على Cloudflare وحده).
3. Commit.
4. **Settings ← Pages** ← Source: **Deploy from a branch** ← `main` / `(root)` ← Save.
5. بعد دقيقة سيظهر الرابط: `https://username.github.io/mudir-thanawiya-2026/`.

### 4) التجربة
افتح الرابط ← تبويب "المساعد الذكي" ← جرّب سؤالاً مثل "ما هي مهام مدير
الثانوية؟" أو "ما عدد المناصب المالية لمسابقة 2026؟".

## ما تم تضمينه في قاعدة المعرفة (ولماذا)
لتبقى الصفحة سريعة التحميل، اخترت:
- ✅ كل الوثائق المرجعية المختصرة (تشريع، علوم التربية، تسيير...).
- ✅ نماذج الامتحانات السابقة القابلة لاستخراج النص (2000-2004 وغيرها).
- ❌ استُبعدت الكتب المرجعية الطويلة (مثل "كتاب المفيد في التربية" أو
  "التحرير الإداري" الكاملين) لأنها تضخّم حجم الصفحة كثيراً (مئات آلاف
  الأحرف لكل كتاب)، وتحتاج فعلياً نظام بحث متجهي (vector search) لا بحثاً
  بسيطاً بالكلمات المفتاحية. هذا متوفر بالفعل في مشروع Dounia EduPass AI
  الكامل الذي بنيناه سابقاً (Cloudflare Vectorize) — يمكن استخدامه متى
  أردت توسيع قاعدة المعرفة هذه لتشمل الكتب الكاملة.
- ❌ أرشيف امتحانات 2006 كان ملفات PDF ممسوحة ضوئياً (صور لا نص)، فلم
  يُستخرج منها نص تلقائياً؛ يمكن إضافته لاحقاً عبر OCR إذا احتجته.

## ملاحظة عن البحث في قاعدة المعرفة
البحث المستخدم هنا (مطابقة كلمات مفتاحية بسيطة في المتصفح) أبسط بكثير من
نظام RAG بالمتجهات (Vectorize) في المشروع الكامل — مناسب جداً لهذا الحجم
من البيانات (984 مقتطفاً) وسريع فوراً، لكنه أقل دقة مع الأسئلة المُعاد
صياغتها بكلمات مختلفة تماماً عن النص الأصلي. كافٍ تماماً للنسخة السريعة
الحالية.
